# SQL Sequences

## Introduction

A SQL Sequence is a database object used to generate unique numeric values automatically. Sequences are commonly used to create primary key values, invoice numbers, order IDs, employee IDs, and other identifiers that must remain unique.

Unlike Auto Increment columns, sequences are independent database objects and can be shared across multiple tables.

## Why Use Sequences?

Sequences provide several advantages:

- Generate unique values automatically.
- Reduce manual ID management.
- Can be shared across multiple tables.
- Support custom starting values and increments.
- Allow both ascending and descending number generation.

## How SQL Sequences Work

A sequence generates numbers based on rules defined during creation. These rules determine:

- The starting value.
- The increment or decrement amount.
- The minimum allowed value.
- The maximum allowed value.
- Whether the sequence restarts after reaching its limit.

Each time a sequence value is requested, the next number is generated automatically.

## Creating a Sequence

### Syntax

```sql
CREATE SEQUENCE sequence_name
START WITH start_value
INCREMENT BY increment_value
MINVALUE minimum_value
MAXVALUE maximum_value
CYCLE;
```

### Parameters

| Parameter | Description |
| --- | --- |
| START WITH | First value generated by the sequence |
| INCREMENT BY | Value added or subtracted each time |
| MINVALUE | Smallest value allowed |
| MAXVALUE | Largest value allowed |
| CYCLE | Restarts the sequence after reaching the limit |
| NOCYCLE | Stops sequence generation at the limit |

## Creating an Ascending Sequence

The following sequence starts at 100 and increases by 1.

```sql
CREATE SEQUENCE employee_sequence
START WITH 100
INCREMENT BY 1
MINVALUE 100
MAXVALUE 10000
CYCLE;
```

### Generated Values

| Request | Generated Value |
| --- | --- |
| First | 100 |
| Second | 101 |
| Third | 102 |
| Fourth | 103 |

## Creating a Descending Sequence

Sequences can also generate values in descending order.

```sql
CREATE SEQUENCE reverse_sequence
START WITH 500
INCREMENT BY -1
MINVALUE 1
MAXVALUE 500
CYCLE;
```

### Generated Values

| Request | Generated Value |
| --- | --- |
| First | 500 |
| Second | 499 |
| Third | 498 |
| Fourth | 497 |

## Creating a Sample Table

```sql
CREATE TABLE TeamMembers (
    MemberID INT PRIMARY KEY,
    MemberName VARCHAR(100),
    Department VARCHAR(50)
);
```

## Using a Sequence During INSERT

Sequence values can be inserted automatically using the `NEXTVAL` keyword.

```sql
INSERT INTO TeamMembers
VALUES (
    employee_sequence.NEXTVAL,
    'Akash Dangudubiyyapu',
    'Management'
);
```

## Getting the Next Sequence Value

A sequence value can be generated without inserting data.

```sql
SELECT employee_sequence.NEXTVAL;
```

Each execution returns the next available number.

## Getting the Current Sequence Value

Some database systems allow access to the most recently generated value.

```sql
SELECT employee_sequence.CURRVAL;
```

This returns the current sequence number for the session.

## Sequence Caching

To improve performance, databases can store sequence values in memory.

### Example

```sql
CREATE SEQUENCE member_sequence
START WITH 1
INCREMENT BY 1
CACHE 20;
```

In this example, 20 sequence values are preallocated in memory, reducing disk operations and improving performance.

## Sequence Without Caching

```sql
CREATE SEQUENCE member_sequence
START WITH 1
INCREMENT BY 1
NOCACHE;
```

This avoids unused cached values but may reduce performance.

## Common Use Cases

### Employee IDs

```sql
CREATE SEQUENCE employee_id_seq
START WITH 1000
INCREMENT BY 1;
```

### Order Numbers

```sql
CREATE SEQUENCE order_number_seq
START WITH 5000
INCREMENT BY 1;
```

### Invoice Numbers

```sql
CREATE SEQUENCE invoice_seq
START WITH 10000
INCREMENT BY 10;
```

## Sequence vs Auto Increment

| Feature | Sequence | Auto Increment |
| --- | --- | --- |
| Independent Object | Yes | No |
| Can Be Shared Across Tables | Yes | No |
| Custom Increment Values | Yes | Limited |
| Supports Descending Values | Yes | Usually No |
| Database Controlled | Yes | Yes |

## Important Points

- Sequences generate unique numeric values automatically.
- They are independent database objects.
- A single sequence can be used by multiple tables.
- Sequence values are generally not reused after generation.
- Cached values may be skipped after unexpected database shutdowns.
- Sequences are commonly used for primary keys and business identifiers.

## Conclusion

SQL Sequences provide a flexible and efficient mechanism for generating unique numeric values. Unlike Auto Increment columns, sequences can be shared across multiple tables and customized with starting values, increments, limits, and cycling behavior. They are widely used in enterprise applications to generate reliable identifiers for records, transactions, orders, and other database entities.
