# Paging in Operating Systems

Paging is a non-contiguous memory management technique that divides a process's logical memory into fixed-size blocks called pages, and splits physical RAM into blocks of the same size called frames. By mapping logical pages to physical frames, paging eliminates the need for contiguous allocation, prevents external fragmentation, and enables efficient virtual memory implementation.

## Basic Principles of Paging

> **Note:** Under paging, the process is divided into equal-sized pages, and physical RAM is divided into frames. This matches page size directly to frame size, eliminating external fragmentation.

Paging addresses critical problems in contiguous memory allocation:

- **Non-Contiguous Loading:** A process does not need to occupy a single continuous block of physical memory. Pages can be scattered across any free frames in physical RAM.
- **On-Demand Execution:** The entire process does not need to be loaded into main memory at once. Only active, required pages are loaded on demand, supporting the execution of programs that exceed the size of physical RAM.
- **Simplified Allocation:** Since memory is managed in uniform, fixed-size pages and frames, the operating system avoids complex compaction routines.

## Core Terminologies

- **Logical Address Space (LAS):** The full set of logical addresses generated by the CPU during the execution of a program.
- **Physical Address Space (PAS):** The total range of hardware addresses available in the physical RAM.
- **Page Table:** A data structure maintained by the operating system kernel for each process, mapping logical page numbers to physical frame numbers.
- **Page Table Entry (PTE):** An individual entry in a page table, containing the target physical frame number and control bits (such as valid-invalid, read/write, and dirty bits).

## Steps in the Paging Cycle

During execution, the operating system and hardware coordinate address mapping through the following steps:

1. **Divide Memory:** The logical memory of a process is divided into pages, and physical RAM is divided into frames of the identical size.
2. **Load Pages:** The OS loads a process's active pages from secondary storage into any available physical frames.
3. **Build Page Table:** The OS maps each logical page number to its allocated physical frame index in the process's page table.
4. **Translate Address:** The CPU's Memory Management Unit (MMU) intercepts every logical reference and uses the page table to translate it into a physical address.
5. **Handle Page Faults:** If the CPU requests a page that is not currently loaded in physical RAM, the system triggers a page fault, pauses execution, retrieves the page from secondary storage, updates the page table, and resumes execution.

## Mathematical Address Calculations

Paging requires precise bit partitioning of logical and physical addresses to map spaces.

### Example 1: Address Bit Size Calculation

Consider a system configured with the following characteristics:

- **Logical Address Space (LAS):** 256 M words (256 * 2^20 = 2^8 * 2^20 = 2^28 words).
- *Logical Address Width:* log2(2^28) = 28 bits.
- **Physical Address Space (PAS):** 64 M words (64 * 2^20 = 2^6 * 2^20 = 2^26 words).
- *Physical Address Width:* log2(2^26) = 26 bits.

### Example 2: Page and Frame Mapping

Consider a system with:

- **Physical Address Width:** 14 bits (Physical Address Space = 2^14 words = 16 K words).
- **Logical Address Width:** 16 bits (Logical Address Space = 2^16 words = 64 K words).
- **Page Size (equal to Frame Size):** 2 K words (2 * 1024 = 2048 words = 2^11 words).

We calculate the indexing layout as follows:

- **Number of Frames:** PAS / Frame Size = 16 K / 2 K = 8 frames (2^3 frames, requiring 3 bits).
- **Number of Pages:** LAS / Page Size = 64 K / 2 K = 32 pages (2^5 pages, requiring 5 bits).
- **Page Offset bits (d):** log2(2 K words) = 11 bits.

### Address Bit Partitioning

The CPU splits the **Logical Address (16 bits)** into two parts:

- **Page Number (p):** The higher-order 5 bits, used as an index to lookup the page table.
- **Page Offset (d):** The lower-order 11 bits, representing the offset within the page.

The MMU translates this into a **Physical Address (14 bits)**:

- **Frame Number (f):** The higher-order 3 bits retrieved from the page table.
- **Frame Offset (d):** The lower-order 11 bits (copied directly from the logical page offset).

The physical address formulation can be expressed using bitwise operators:

```text
Physical Address = (Frame Number << Number of Bits in Frame Offset) + Frame Offset
```

## Hardware Implementation and the Translation Lookaside Buffer (TLB)

Because page tables are large, they are stored in the computer's main memory (RAM). However, this introduces a performance issue: every logical memory access requires **two physical memory accesses** (one to read the page table entry to locate the frame, and one to read/write the target data).

```text
Without TLB:
Effective Memory Access Time = m (Page Table Read) + m (Target Page Access) = 2m
(where m is the physical memory access latency)
```

To resolve this latency, computer hardware incorporates a **Translation Lookaside Buffer (TLB)** inside the MMU. The TLB is a small, fast, associative hardware cache.

### Working of the TLB

- **Associative Search:** Each entry in the TLB stores a tag (page number) and a value (frame number). When the CPU generates an address, the TLB compares the page number with all tags in parallel.
- **TLB Hit:** If the page number is in the TLB, the frame number is returned instantly, avoiding the memory access to the page table.
- **TLB Miss:** If the page number is not in the TLB, the MMU reads the page table in main memory, updates the TLB, and accesses the target frame.

## Summary

Paging is a memory management technique that splits processes into pages and memory into frames of the same size. By mapping logical pages to non-contiguous frames via a page table, paging resolves the problem of external fragmentation and supports virtual memory execution. To mitigate the double-memory-access latency introduced by page tables, modern processor architectures utilize a hardware cache called the Translation Lookaside Buffer (TLB) to perform translations quickly.
